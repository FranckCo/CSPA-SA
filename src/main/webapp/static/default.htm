Implementing CSPA Project 2014

Seasonal Adjustment Service

The Common Statistical Production Architecture (CSPA) was developed during 2013 in an international collaboration project under the High-Level Group for the Modernisation of Statistical Production and Services (HLG). That project included the development and specification of the architecture, as well as a practical test of its principles and applicability in a "proof of concept". However, given the strict time limits for that project (one year), it deliberately did not include the implementation of the architecture for production systems within statistical organisations. A follow-up project to cover early implementations and the development of key supporting resources such as a catalogue of artefacts, was selected by the HLG as one of two key modernisation projects for 2014.

The Seasonal Adjustment Service (SAS) is one of the CSPA services developed in the "Implementing CSPA" Project. It is a REST service wrapping the JDemetra+ software.

The following operations are available:

In this table, the URI are relative, and it is assumed that 'sas' is the web context for the service.
HTTP methodURIDescription
GET/sas, /sas/ts, /sas/x13Redirects to a HTML page describing the service (this page).
GET/sas/ts/{id}Returns an XML file describing the predefined configuration {id} for Tramo-Seats.
GET/sas/x13/{id}Returns an XML file describing the predefined configuration {id} for X13.
